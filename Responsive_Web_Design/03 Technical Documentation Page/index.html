コマンド一覧
git init git add git commit git push git pull git status git log git branch git checkout git merge

git init

新しいGitプロジェクトを始めるためのコマンドです。

このコマンドを実行すると、カレントディレクトリに.gitという名前のディレクトリが作成されます。

このディレクトリには、Gitリポジトリの設定と、リポジトリの変更履歴などのデータが保存されます。

具体的には、次のように使用されます:

git init

git add

変更したファイルをステージングエリアに追加します。

使い方は次の通りです：

# 単一のファイルをステージングする git add [file_name] # 変更されたすべてのファイルをステージングする git add . # 特定のディレクトリ内の変更をステージングする git add [directory_name]

git commit

ステージングエリアにある変更をローカルリポジトリにコミットするためのコマンドです。

基本的な使用方法は次のとおりです：

git commit -m "コミットメッセージ"

オプションについては、以下があります：

  • -a: 変更されたファイルをステージングせずに、変更を含む全てのトラック対象ファイルをコミットします。
  • -m: コミットメッセージを直接コマンドラインから入力します。
  • -am: -a オプションと -m オプションを一緒に使用して、ステージングとコミットを同時に行います。

git push

ローカルのコミットをリモートリポジトリにアップロードするためのコマンドです。

基本的な使用方法は以下の通りです：

git push [リモート名] [ブランチ名]

このコマンドは、指定したブランチの変更をリモートに送信します。

[リモート名] は通常 origin が使われますが、プロジェクトによっては異なる場合があります。

[ブランチ名] は通常、現在のブランチ名です。

オプションについては、以下があります：

  • -u: 'git push -u origin main' のように使用します。これにより、次回の git push でブランチ名を指定しなくても済むように、トラッキング情報が設定されます。
  • --force (または -f): 強制プッシュです。これを使用すると、リモートの履歴と合わない場合にも変更を強制的にプッシュできますが、注意が必要です。履歴が消去されたり、他の開発者の変更を上書きしてしまう可能性があります。

git pull

リモートリポジトリから最新の変更を取得し、現在のブランチにマージするためのコマンドです。

基本的な使用方法は次の通りです：

git pull [リモート名] [ブランチ名]

これは、指定したリモートリポジトリから指定したブランチの最新の変更を取得して、ローカルの対応するブランチにマージします。

例えば:

git pull origin main

これは、origin という名前のリモートリポジトリから main ブランチの最新の変更を取得し、ローカルの main ブランチにマージします。

オプションについては、以下があります：

  • --rebase: git pull --rebase origin main のように使用します。このオプションは、git pull を実行した後にローカルの変更をリモートの変更の後ろに追加します。これにより、クリーンで直感的な履歴が維持されることがあります。
  • --ff-only: git pull --ff-only origin main のように使用します。これは、git pull を実行する際に、Fast-forwardマージだけを許可します。これにより、自動的にマージが行われ、コンフリクトを避けることができます。

git status

現在の作業ディレクトリの状態を確認するためのコマンドです。

変更されたファイルやステージングエリアにあるファイル、およびコミットされていないファイルを表示します。

基本的な使用方法は次の通りです：

git status

オプションについては、以下があります：

  • --short（または -s）: git status -s のように使用します。短縮された状態で、ファイルの状態をより簡潔に表示します。変更されたファイル、ステージングエリアにあるファイル、および未追跡のファイルを一覧表示します。
  • --branch（または -b）: git status -b のように使用します。現在のブランチ名や追跡ブランチなど、ブランチに関する詳細な情報も表示されます。
  • --untracked-files（または -u）: git status -u のように使用します。このオプションを使用すると、未追跡のファイルも表示されます。

git log

リポジトリのコミット履歴を表示するためのコマンドです。

このコマンドを使用すると、リポジトリの過去のコミットに関する情報を見ることができます。

基本的な使用方法は次の通りです：

git log

このコマンドは、コミット履歴を時系列順に表示します。コミットごとに、コミットID、著者、日付、コミットメッセージなどの情報が表示されます。

オプションについては、以下があります：

  • --oneline: git log --oneline と使用します。短い形式でコミットを表示します。各コミットを1行で表示し、コミットIDとコミットメッセージが表示されます。
  • -n [num]: git log -n 5 のように使用します。表示するコミットの数を制限します。この場合は最新の5つのコミットのみ表示されます。
  • --graph: git log --graph と使用します。コミットの履歴をグラフで表示します。マージやブランチの分岐など、コミット間の関係性を視覚的に表示します。
  • --since および --until: git log --since="2 weeks ago" --until="1 week ago" のように使用します。指定した日付範囲内のコミットを表示します。
  • --author: git log --author="author_name" のように使用します。特定の著者が行ったコミットのみを表示します。

git branch

ブランチに関する情報を表示・管理するためのコマンドです。

リポジトリ内のブランチの一覧を表示したり、新しいブランチを作成したり、現在のブランチを変更したりするのに使用されます。

基本的な使い方は次の通りです：

  • ブランチ一覧を表示する: git branch

    • このコマンドは、ローカルリポジトリ内のすべてのブランチを一覧表示します。現在のブランチは、名前の前に * として表示されます。

  • 新しいブランチを作成する: git branch [branch_name]

    • このコマンドは、現在のブランチから新しいブランチを作成しますが、まだ新しいブランチに切り替えたわけではありません。

オプションについては、以下があります：

  • -d : git branch -d と使用します。指定されたブランチを削除します。
  • -r : git branch -r と使用します。リモートリポジトリのブランチを表示します。
  • -a : git branch -a と使用します。ローカルとリモートのすべてのブランチを表示します。
  • -m : git branch -m [new_branch_name] と使用します。現在のブランチの名前を変更します。
  • --merged : git branch --merged と使用します。マージされたブランチの一覧を表示します。
  • --no-merged : git branch --no-merged と使用します。まだマージされていないブランチの一覧を表示します。

git checkout

ブランチの切り替えやファイルの復元に使用されます。

基本的な使い方は次の通りです：

  • ブランチの切り替え: git checkout [branch_name]

    • このコマンドを使用すると、指定されたブランチに移動します。

  • 特定のファイルの復元: git checkout [file_name]

    • これにより、最新のコミットの状態にそのファイルを復元します。

  • 特定のコミットからファイルを取得: git checkout [commit_hash] [file_name]

    • これにより、指定したコミットからファイルを取得できます。

オプションについては、以下があります：

  • -b : 既存のブランチから新しいブランチを作成して切り替えます。
  • -f : 強制的に変更を行います。未コミットの変更は失われる可能性がありますので、注意が必要です。
  • -- : ファイルを直前のコミット時点の状態に戻すためのコマンドです。

git merge

異なるブランチの変更を統合するために使用されます。

通常は、変更を取り込むための「対象ブランチ」を現在のブランチに統合します。

基本的な使用法は次のとおりです：

git merge [branch_name]

これにより、指定した [branch_name] の変更が現在のブランチにマージされます。

オプションについては、以下があります：

  • --ff: デフォルトの Fast-forward マージを強制します。
  • --no-ff: Fast-forward　を禁止してマージを行います。このオプションを使用すると、マージコミットが作成され、履歴にマージが行われたことが明示されます。
  • --squash: マージするコミットを1つにまとめます。複数のコミットを1つの大きなコミットにまとめたい場合に使用されます。
  • --abort: マージの途中で何らかの問題が発生した場合、マージを取り消し、マージ前の状態に戻ります。
